# MERN User Management App with Docker

A simple full-stack user management application built with the MERN stack and containerized with Docker Compose. The app lets you create users, list all stored users, and persist the data in MongoDB.

## Tech Stack

- **Frontend:** React 19, Vite, React Router, Axios
- **Backend:** Node.js, Express 5, Mongoose
- **Database:** MongoDB
- **Development:** Nodemon, ESLint
- **Containers:** Docker and Docker Compose

## Features

- Create a user with a name and email address
- Display all users stored in MongoDB
- Refresh the user list from the UI
- Basic loading, success, and error states
- Unique email validation at the database level
- MongoDB data persistence through a Docker volume
- Hot reload support for frontend and backend containers

## API Endpoints

Base URL:

```text
http://localhost:5001/api/users
```

| Method | Endpoint | Description |
| --- | --- | --- |
| `GET` | `/api/users` | Get all users |
| `POST` | `/api/users` | Create one or more users |

The POST endpoint currently expects an **array of users**, even when creating only one user.

Example request:

```json
[
  {
    "name": "Ravi Sharma",
    "email": "ravi@example.com"
  }
]
```

Example with cURL:

```bash
curl -X POST http://localhost:5001/api/users \
  -H "Content-Type: application/json" \
  -d '[{"name":"Ravi Sharma","email":"ravi@example.com"}]'
```

## Run with Docker

### Prerequisites

- Docker Desktop
- Docker Compose

The Compose setup exposes:

| Service | Port |
| --- | --- |
| React frontend | `5173` |
| Express backend | `5001` |
| MongoDB | `27017` |

Before starting the containers, note that the current frontend `Dockerfile` command uses an invalid npm command:

```dockerfile
CMD ["npm", "start", "run", "dev", "--", "--host"]
```

Change it to:

```dockerfile
CMD ["npm", "run", "dev", "--", "--host", "0.0.0.0"]
```

The backend command can similarly be simplified to:

```dockerfile
CMD ["npm", "start"]
```

Then build and start the complete stack:

```bash
docker compose up --build
```

Open:

```text
Frontend: http://localhost:5173
Backend:  http://localhost:5001/api/users
```

Run containers in the background:

```bash
docker compose up --build -d
```

View logs:

```bash
docker compose logs -f
```

Stop the containers:

```bash
docker compose down
```

Stop the containers and delete MongoDB data:

```bash
docker compose down -v
```

> `docker compose down -v` permanently removes the `mongo-data` volume and all users stored in it.

### Prerequisites

- Node.js 20 or newer

```bash

npm start
```

The backend runs at `http://localhost:5001`.

Open another terminal:

```bash
npm run dev
```

The frontend runs at `http://localhost:5173`.

## Data Model

Each user document has the following fields:

| Field | Type | Rules |
| --- | --- | --- |
| `name` | String | Required |
| `email` | String | Required and unique |
| `_id` | ObjectId | Generated by MongoDB |

## Troubleshooting

### Frontend cannot reach the API

- Confirm that the backend is running on port `5001`.
- Confirm that `client/src/services/api.js` points to `http://localhost:5001/api/users`.
- The backend currently allows CORS requests from `http://localhost:5173`.

### Backend cannot connect to MongoDB

- With Docker, use `mongodb://mongo:27017/mern-docker`.
- Without Docker, use `mongodb://localhost:27017/mern-docker`.
- Check MongoDB logs with `docker compose logs mongo`.

### Email already exists

The `email` field is unique. Use a different email address or remove the existing record from MongoDB.

### Changes are not detected inside Docker

Polling is enabled through `CHOKIDAR_USEPOLLING` and `WATCHPACK_POLLING`. Rebuild the containers if dependency or Dockerfile changes are not reflected:

```bash
docker compose up --build
```

## Notes

- The frontend API URL and backend CORS origin are currently hard-coded for local development.
- The current UI provides create and list operations. Update and delete API routes are not implemented.
- This setup is development-oriented and uses bind mounts plus hot reload.
